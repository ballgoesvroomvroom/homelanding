Caveats

  • Complex exponents (e.g. 10^(x + 1)) is unable to be multiplied by another term, e.g. 10^(x + 1) *3, will produce inaccurate result
  • Double or more complex chained exponents (e.g. 10^(10^x)); 10^(3^(2^2)) is still okay, they are not algebraic terms
  • Despite being same base, machine is unable to mash them together, e.g. 10^x * 10^x = 10^2x (engine cannot do this)
  • Multiplication order should not matter, I have tested engine's ability to reorder coefficients and constants before variables (to a certain degree); please help to test e.g. x^3 *3

Test cases

  • x^3 *3(10-8)
  • (7 + 3)
  • x^2 * 3x ^4
  • 10^x * 10 ^x
